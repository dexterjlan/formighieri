Modo teste vs produção

Desenvolvimento (config.dev.js): NOTIFICATION_TEST_MODE: true — todos os e-mails vão para dexterjl@gmail.com.
Produção (config.prod.js): NOTIFICATION_TEST_MODE: false — destinatários conforme regras abaixo.

Atenção: nos fluxos de Aprovação Comercial e Requisição, o campo Para (TO) está fixo em NOTIFICATION_TEST_EMAIL no código, mesmo em produção. Os destinatários reais de negócio recebem via CC configurado em Gestão → Configurações do Sistema.

Glossário de destinatários

Grupo Regra no banco (appUsers)
Gestores (todos)
  • Admin ou Consultor com gestorComercial = true
  • Admin ou Projetista com gestorProjetos = true
  • Marceneiro com gestorFabrica = true
Todos devem estar isActive = true.
Gestores comerciais Admin ou Consultor com gestorComercial = true e isActive = true
Gestores de projetos Admin ou Projetista com gestorProjetos = true e isActive = true
Projetistas PPCP role = Projetista, ppcp = true, isActive = true
Conferentes role = Projetista, conferente = true, isActive = true
Compras role = Compras, isActive = true
Consultor do pedido E-mail do usuário cujo nome coincide com consultantName do pedido em salesOrders
Projetista responsável E-mail do designerId do projeto em OrderProject

Configuração (cópia — CC)

Campo Tela Uso
E-mails em cópia — Aprovações Gestão → Configurações do Sistema CC em todos os e-mails de aprovação comercial (solicitação, aprovação, revisão, reenvio)
E-mails em cópia — Requisições Gestão → Configurações do Sistema CC em todos os e-mails de requisição técnica (criação e resposta)
Valores armazenados em SystemSettings (chave default). Lista separada por vírgula ou ponto-e-vírgula.
